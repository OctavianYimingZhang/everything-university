# Announcement Sources

Use this reference to collect communications with minimal manual input from the user.

## Source Priority

1. LMS announcements API or announcements page.
2. LMS inbox/messages/notification center.
3. Course pages that list updates or weekly notices.
4. University email web interface after user login, when allowed.
5. User-configured email export, forwarding, or local downloaded `.eml` files.
6. User-pasted announcement text only when official access is unavailable.

## Email Constraint

University email often has no available MCP connector. Treat LMS announcements and notification centers as the primary source. Use webmail only through the user's own authenticated browser session or an approved export.

## Action Extraction

Extract action items from:

- deadline changes;
- new required preparation;
- changed room, time, or delivery mode;
- new quiz, assignment, or lab instruction;
- feedback release notices;
- policy changes;
- office-hour changes.

Keep each action linked to the original announcement id.
